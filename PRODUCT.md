# Agnes Image Studio 产品文档

## 产品概览

Agnes Image Studio 是一个面向 AI 图片与短视频生成的 Web 工作台。当前版本提供图片生成、视频生成、任务队列、本地历史、预览下载、Prompt 优化、图片理解、主题语言切换、健康状态提示和基础合规页面。

产品以浏览器前端为主要操作界面，通过后端 API 提交生成任务、查询任务进度、下载生成结果，并在浏览器 `localStorage` 中保存偏好、草稿和本地历史索引。

当前实现不包含账号系统、支付、分享、云相册、团队协作、统计追踪、在线素材库或权限管理能力。

## 目标用户

- 需要快速生成图片概念图、产品视觉、风格转换图或多图合成图的创作者。
- 需要基于文字、参考图、多图或关键帧生成短视频的运营、设计和内容生产人员。
- 需要在投放或发布前查看数据处理说明、使用限制和联系方式的审核或维护人员。

## 信息架构与路由

路由由 `web/src/routes.tsx` 定义，所有页面挂载在 `RootLayout` 下：

- `/`：首页，介绍产品定位、核心功能、使用流程和透明披露。
- `/image`：图片生成页，提供文生图、图生图、多图合成。
- `/video`：视频生成页，提供文生视频、图生视频、多图视频、关键帧动画。
- `/privacy`：隐私政策页。
- `/terms`：服务条款页。
- `/contact`：联系页。
- `*`：404 页面，提供返回首页、图片生成页和视频生成页的入口。

`RootLayout` 注入全局偏好 Provider 和任务队列 Provider。`App` 负责站点外壳、页面元信息、顶部导航、设置面板、健康状态、任务抽屉和页脚。

## 全局页头、页脚与设置

页头包含品牌入口、主导航和任务按钮。主导航当前只有首页、图片生成、视频生成三个入口；隐私政策、服务条款和联系页通过页脚进入。任务按钮显示当前进行中任务数与最大并发上限，点击后打开右侧任务抽屉。

设置面板在页头中提供：

- 主题：支持 `auto`、`default`、`light`、`dark`，会写入 `agnes:theme`，并同步 `data-theme`、`color-scheme` 和 `theme-color`。
- 语言：支持中文与英文切换，写入 `agnes:language`，同时更新 `document.documentElement.lang`。
- 健康状态灯：每 5 秒调用 `/api/health`。后端可用且配置 API Key 时显示正常状态；未配置 API Key 或后端不可用时显示警告状态，并通过 `title`/`aria-label` 表达具体原因。

页脚展示品牌名、AI 生成工具披露文本，并提供隐私政策、服务条款、联系页链接。在图片和视频生成页会使用更宽的页脚布局。

## 首页

首页由 `HomePage` 实现，面向新用户说明产品功能和使用流程：

- Hero 区展示产品定位和两个主要行动入口：开始生成图片、创建视频。
- 功能卡片介绍 AI 图片生成、AI 视频生成、历史预览下载工作流，并链接到对应页面或隐私说明。
- 使用流程以四步说明：输入 prompt、上传有权使用的图片或粘贴公网 URL、选择生成参数、等待完成并下载结果。
- 透明披露说明 AI 输出可能不准确或不适合发布，用户需确保素材权利，服务不保证固定生成时间、固定质量、广告审核结果或商业适用性。

## 图片生成页

图片生成页由 `ImageGenerate` 实现，左侧是输入与参数面板，右侧是本地历史。

支持三种图片模式：

- 文生图 `text2img`：只需要 prompt，可选择尺寸和输出格式。
- 图生图 `img2img`：需要 1 张参考图片，可上传本地图片或粘贴图片 URL。
- 多图合成 `multi`：需要至少 2 张参考图片，最多 8 张。

主要功能：

- Prompt 输入：使用多行文本框，按当前模式显示不同占位示例。
- Prompt 优化：点击 `✦` 按钮调用 `/api/text/optimize-prompt`，按当前生成模式优化提示词并回填；空 prompt 会提示先输入。
- 图片理解：在图生图或多图模式下显示。它读取第一张本地图片或 URL，调用 `/api/text/understand-image`，生成符合当前模式的提示词；结果支持填入 prompt 和复制。多图时界面提示只分析第一张。
- 上传：支持本地图片和公网图片 URL。本地图片会用于预览、提交和草稿保存；URL 输入不强制前端校验协议，但界面提示使用 `https://` 公网 URL。
- 尺寸：使用统一尺寸预设，默认 `1024x768`，可重置默认值。图片尺寸偏好写入 `agnes:image-size`。
- 输出格式：支持 `url` 和 `b64_json`。
- 提交：调用任务队列提交 `/api/generate`。必须有非空 prompt；图生图和多图模式还需满足图片数量；队列满或节流锁定时按钮禁用。
- 提交成功后：显示短暂接受反馈，清空 prompt、参考图和图片草稿，保留尺寸偏好，并触发历史刷新。
- 历史回填：点击历史记录可回填模式、prompt、尺寸、输出格式和历史中保存的参考图 URL；本地文件不会从历史中回填。

## 视频生成页

视频生成页由 `VideoGenerate` 实现，结构与图片页一致：左侧输入与参数，右侧本地历史。

支持四种视频模式：

- 文生视频 `text2vid`：只需要 prompt。
- 图生视频 `img2vid`：需要 1 张参考图片。
- 多图视频 `multivid`：需要至少 2 张图片，最多 8 张。
- 关键帧动画 `keyframes`：需要至少 2 张关键帧图片，最多 8 张。

主要功能：

- Prompt 输入：多行文本框，并提供 Prompt 优化按钮。
- 图片理解：非文生视频模式下显示，可基于第一张参考图生成提示词。
- 负向提示词：可选，用于描述要避免的内容，写入视频偏好。
- 时长预设：`81`、`121`、`241`、`441` 帧，对应约 3 秒、5 秒、10 秒、18 秒。
- 帧率：支持 `8`、`12`、`16`、`24`、`30` FPS。
- 分辨率：复用图片尺寸预设，默认 `1024x768`，可重置默认值。
- 随机种子：可选；留空时不提交 seed。
- 预计时长：根据帧数和帧率计算展示。
- 提交：调用任务队列提交 `/api/videos`。非文生视频模式会提交本地文件和图片 URL；后端可进一步处理本地图片上传。
- 提交成功后：清空 prompt、上传文件、图片 URL 和视频草稿；视频模式、负向提示词、帧数、帧率、尺寸、seed 等偏好继续保留。
- 历史回填：可回填视频模式、prompt、负向提示词、帧数、帧率、尺寸、seed 和参考图 URL。
- 视频播放：任务抽屉和本地历史中的已完成视频会以内嵌 `<video controls>` 播放器展示；本地历史还提供预览弹窗入口，任务抽屉视频结果提供新标签打开原始视频入口。
- 视频播放：任务抽屉和本地历史中的已完成视频会以内嵌 `<video controls>` 播放器展示；本地历史还提供预览弹窗入口，任务抽屉视频结果提供新标签打开原始视频入口。

页面文案明确说明视频任务耗时受队列、视频长度和上游服务状态影响，发布前需要人工审查。

## 隐私政策页

隐私政策页说明当前实现会处理的数据类型和用途：

- 处理 prompt、生成参数、上传文件、图片 URL、任务状态和生成结果。
- 浏览器使用 `localStorage` 保存偏好、草稿和本地历史。
- 数据用于提交 AI 生成任务、展示任务进度、预览下载结果、保存历史记录和维护服务稳定性。
- 图片和视频生成请求会发送给 Agnes API；视频模式中的本地图片可能先上传到 ImgBB 转换为公网 URL。
- 当前版本不加入 Google Ads、Analytics 或 Tag Manager 追踪脚本。
- 用户可清空浏览器本地历史和草稿，并应避免上传无权使用、违法、敏感或侵权内容。

## 任务队列

任务队列由 `TaskQueueProvider` 和 `useMultiGeneration` 实现，是图片和视频提交的统一入口。

关键行为：

- 最大进行中任务数为 3。提交中、排队中和生成中的任务计入上限；提交失败不计入。
- 全局提交节流为 1200ms。节流期间生成按钮显示提交中或禁用状态。
- 任务以本地 `localId` 记录，同时保存 prompt 摘要、类型、尺寸、服务端任务对象、提交错误和重试状态。
- 提交后立即在任务抽屉中出现，并独立轮询服务端任务状态。
- 轮询间隔为 1200ms；查询到 `done` 或 `error` 后停止轮询。
- 服务端队列位置会从任务 debug events 中读取并显示。
- 每个任务卡可移除；提交后查询失败且已有服务端任务 ID 时可重试查询。
- 完成或失败的任务会写入本地历史。

右侧任务抽屉集中展示所有当前标签页内的任务。任务卡展示类型、prompt 摘要、尺寸、提交阶段错误、排队位置、进度、结果和移除按钮。

## 本地历史

本地历史由 `TaskHistory` 和 `localHistory` 实现，以浏览器 `localStorage` 为索引存储，键名为 `agnes:task-history:v1`。

能力包括：

- 图片和视频历史按分类展示，各分类最多读取 100 条，总量最多保留 200 条。
- 完成、失败和进行中任务都会随着轮询结果写入历史。
- 页面打开或提交后会读取本地历史；如果存在进行中任务，会自动调用 `/api/tasks/:id` 刷新状态。
- 进行中历史记录每 1500ms 轮询刷新。
- 如果历史中的进行中任务查询失败，会标记为本地过期错误。
- 可删除单条历史记录。
- 图片失败记录提供刷新按钮，调用图片任务刷新接口后重新查询。
- 历史记录点击可回填输入；当前输入已有内容时会弹窗确认，避免误覆盖。

历史工具栏提供：

- 记录数展示。
- 搜索：按类型、状态、prompt、负向 prompt、尺寸、宽高、秒数、视频 URL、视频 ID、修订 prompt、图片 URL 等文本匹配。
- 状态筛选：全部、进行中、已完成、失败。
- 排序：最新优先、最早优先。
- 布局：列表和瀑布流。

这些历史工具栏偏好按图片和视频分类分别持久化，例如 `agnes:image-history-search`、`agnes:video-history-sort`、`agnes:image-history-layout`。

## 上传与预览

上传组件支持：

- 点击按钮选择本地图片，文件输入接受 `image/*`。
- 粘贴公网图片 URL 并添加。
- 展示已选数量和最大数量。
- 删除单个本地文件或 URL。
- 点击缩略图打开预览弹窗。
- 可配置单文件大小上限，超过上限时跳过并显示提示；当前图片页和视频页没有传入额外大小上限。

预览弹窗支持图片和视频：

- 点击遮罩或关闭按钮关闭。
- 按 `Escape` 关闭。
- 图片以 `<img>` 展示，视频以带 controls 的 `<video>` 展示。
- 弹窗底部提供普通链接下载入口和关闭按钮。
- 该下载入口是普通 `<a>` 链接，不显示 Blob 下载 loading。

## 下载

当前有两类下载实现：

- 任务抽屉中的图片结果仍使用普通链接跳转到 `/api/tasks/:id/download/:index`。
- 视频结果和历史记录中的图片/视频结果使用 `BlobDownloadButton`，通过 `fetch -> Blob -> objectURL -> a.click()` 实现无刷新下载，并只在当前点击的按钮显示 loading 状态。
- 预览弹窗中的下载入口沿用普通 `<a>` 链接。
- 预览弹窗中的下载入口是普通 `<a>` 链接，不显示 Blob 下载 loading。

文件名规则：

- 历史图片：`agnes-image-{taskId}-{index}.png`。
- 历史视频：`agnes-video-{taskId}.mp4`。
- 任务抽屉视频：`agnes-video-{taskId}.mp4`。

## Prompt 优化与图片理解

Prompt 优化调用 `/api/text/optimize-prompt`，请求体包含 `prompt` 和当前模式 `mode`。成功后用返回的 prompt 替换输入框内容，并短暂显示完成状态；失败时显示错误。

图片理解调用 `/api/text/understand-image`，请求体包含图片、当前模式和可选 prompt。图片来源优先使用第一张本地文件的 Data URI，其次使用第一条 URL。结果展示在面板内，支持填入提示词和复制到剪贴板。复制失败时不打断用户操作。

## 健康状态与 API 客户端

前端 API 客户端封装了：

- `/api/generate`：提交图片任务。
- `/api/videos`：提交视频任务。
- `/api/tasks/:id`：查询任务。
- `/api/tasks?category=&limit=`：获取任务列表，当前前端历史主要使用本地历史读取。
- `/api/tasks/:id/refresh`：刷新图片任务。
- `/api/videos/:id/refresh`：刷新视频任务。
- `/api/tasks/:id/download/:index`：下载图片。
- `/api/videos/:id/download`：下载视频。
- `/api/text/optimize-prompt`：优化 prompt。
- `/api/text/understand-image`：图片理解。
- `/api/health`：健康状态。

429 响应会被封装为 `RateLimitError`，携带分类、重试时间和限制数量，用于前端显示“请求过于频繁”的友好提示。其他非 2xx 响应会转为普通错误。

健康状态返回 `ok`、`apiKeyConfigured`、模型、队列状态和可选限流快照。当前 UI 只以状态灯和 tooltip 形式呈现 API Key 或后端状态。

## 通知

通知工具位于 `web/src/utils/notify.ts`。当前页面加载时会初始化浏览器通知状态；浏览器不支持、拒绝授权或权限不可用时会记录状态并停止发送。

首次打开页面且本地没有持久化通知状态、浏览器权限仍为 `default` 时，初始化逻辑会调用 `Notification.requestPermission()`；如果浏览器返回 `granted`，会写入 `agnes:notifications-enabled=1` 和 `agnes:notification-permission-status=granted`。

首次打开页面且本地没有持久化通知状态、浏览器权限仍为 `default` 时，初始化逻辑会调用 `Notification.requestPermission()`；如果浏览器返回 `granted`，会写入 `agnes:notifications-enabled=1` 和 `agnes:notification-permission-status=granted`。

任务轮询到完成或失败后，会尝试发送桌面通知。发送条件是浏览器支持 Notification、权限为 `granted`，并且本地通知开关状态为启用。通知内容包含任务类型、完成时间、prompt 摘要或失败原因；成功的图片任务会尝试使用首张结果图作为通知 icon。点击通知会聚焦回当前页面。

当前页面组件中没有独立可见的通知开关按钮；通知能力主要由初始化状态和任务完成回调驱动。

## 草稿、偏好与持久化

浏览器端持久化集中使用 `localStorage`，统一使用 `agnes:` 前缀。

草稿：

- 图片草稿：`agnes:image-draft`，保存模式、prompt、尺寸、输出格式和 URL 列表。
- 图片草稿文件：`agnes:image-draft-files`，将本地文件序列化为 Data URI，恢复时还原为 `File`。
- 视频草稿：`agnes:video-draft`，保存模式、prompt、负向 prompt、帧数、帧率、尺寸、seed 和 URL 列表。
- 视频草稿文件：`agnes:video-draft-files`，同样序列化本地文件。
- 生成提交成功后，当前页面会清空对应草稿，为下一次输入准备空状态。

偏好：

- 主题：`agnes:theme`。
- 语言：`agnes:language`。
- 图片尺寸：`agnes:image-size`。
- 视频尺寸：`agnes:video-size`。
- 视频模式：`agnes:video-mode`。
- 视频帧数：`agnes:video-frames`。
- 视频帧率：`agnes:video-frame-rate`。
- 视频负向 prompt：`agnes:video-negative-prompt`。
- 视频 seed：`agnes:video-seed`。
- 历史搜索、状态筛选、排序、布局：按图片和视频分类分别保存。
- 通知状态：`agnes:notifications-enabled` 和 `agnes:notification-permission-status`。

所有本地存储读写都捕获异常，隐私模式、配额不足或存储不可用时不会中断主流程。

## 限制与异常状态

当前产品限制：

- 进行中任务最多 3 个，生成提交有 1200ms 全局节流。
- 视频页文案说明每分钟最多 20 次请求；实际限流错误由后端 429 响应返回并显示重试时间。
- 图片模式中图生图最多 1 张参考图，多图合成最多 8 张；视频模式中图生视频最多 1 张参考图，多图视频和关键帧动画最多 8 张。
- 多图图片理解只分析第一张图片。
- 历史回填只恢复历史中保存的 URL 参考图，不恢复本地上传文件。
- 本地历史依赖浏览器 `localStorage`，换浏览器、清理缓存或存储失败会丢失本地索引。
- 联系邮箱仍是占位地址，上线前必须替换。
- 隐私页明确当前没有 Google Ads、Analytics 或 Tag Manager 追踪脚本。
- 没有账号、支付、分享、云相册、团队协作、统计追踪等能力。

异常状态：

- 后端离线或健康检查失败：健康状态灯显示警告。
- 未配置 API Key：健康状态灯显示警告。
- 任务队列满：生成按钮显示队列已满并禁用。
- 图片数量不足：按钮显示还需几张图片。
- prompt 为空：生成按钮禁用；Prompt 优化会提示先输入提示词。
- 提交频率超限：显示每分钟限制和约需等待秒数。
- 上游繁忙、503、429、可重试错误：结果区显示服务繁忙提示和降低尺寸或错峰重试建议。
- 非可重试错误：结果区显示用户友好错误消息，必要时附带后端错误数据。
- 完成但没有图片或视频 URL：结果区显示未返回图片的错误文案。
- 历史查询失败：进行中历史会被标记为任务不存在或已过期。
- Blob 下载失败：弹窗显示 HTTP 状态或错误消息。

空状态：

- 本地历史加载中显示加载提示；没有本地历史记录时显示空历史提示。
- 搜索或筛选后没有匹配记录时显示无匹配提示。
- 任务抽屉中的任务提交后、服务端任务返回前显示提交中状态。
- 结果区在任务未完成前不渲染图片或视频结果；完成但缺少结果 URL 时显示错误提示。

空状态：

- 本地历史加载中显示加载提示；没有本地历史记录时显示空历史提示。
- 搜索或筛选后没有匹配记录时显示无匹配提示。
- 任务抽屉中的任务提交后、服务端任务返回前显示提交中状态。
- 结果区在任务未完成前不渲染图片或视频结果；完成但缺少结果 URL 时显示错误提示。

## 维护提示

- 修改前端页面或组件后，应运行 `npm --prefix web run lint` 和 `npm run build`。
- 修改页面功能时，应同步检查本文件、隐私政策、服务条款和首页披露是否仍准确。
- 修改上传、任务队列、下载或通知逻辑时，应特别验证移动端布局、Blob 下载 loading、任务并发上限和 `localStorage` 键名兼容性。
- 上线前必须替换联系页占位邮箱，并确认后端环境变量、ImgBB 配置和 Agnes API Key 已正确配置。
