# Agnes Image Studio UI 设计规划

## 1. 产品体验定位

Agnes Image Studio 是面向图片与短视频生成的轻量 Web 工作台。体验重点不是内容社区或资产管理，而是让用户在一个浏览器页面内快速完成输入、上传、参数选择、提交、等待、预览、下载和本地历史回填。

当前能力边界明确：不规划账号、支付、分享、云相册、团队协作、统计追踪或权限管理。所有设计都围绕已实现的路由、后端 API、任务队列和浏览器本地持久化展开。

核心体验目标：

- 主功能优先：生成输入、上传、参数、提交按钮和结果历史始终占据页面主空间。
- 极简表达：移除冗余引导、队列解释、状态说明和免责声明式文案，只保留完成操作所需的信息。
- 本地工作台：草稿、偏好、历史筛选、历史布局和通知偏好使用 `localStorage` 维持连续体验。
- 可恢复：用户可从任务抽屉查看当前标签页任务，也可从本地历史找回已完成、失败或进行中的任务。
- 可落地：规划必须贴合现有 `web/src/App.tsx`、`ImageGenerate`、`VideoGenerate`、`TaskHistory`、`TasksDrawer`、`Uploader`、`MediaPreviewModal` 和 `BlobDownloadButton`。

## 2. 设计原则

1. 生成路径短：进入图片或视频页后，首屏优先看到模式、上传、Prompt、参数和生成按钮。
2. 历史是生产辅助，不抢主流程：桌面端保留右侧历史；移动端历史下沉到输入之后，避免压缩表单。
3. 信息分层克制：常态只显示当前值、必要错误和可执行按钮；详细原因放入 `title`、`aria-label` 或错误展开区域。
4. 状态表达轻量：健康状态用状态灯；任务进度用进度条和短标签；提交成功用短暂反馈。
5. 文案去重复：`PRODUCT.md` 中重复的视频播放、预览下载、通知和空态说明在 UI 文案中只保留一处。
6. 触控优先：所有按钮、输入框、选择器、关闭按钮和图标按钮高度不少于 44px。
7. 移动端无横向溢出：320px-480px 宽度下所有布局单列、工具栏可换行、长 URL 和 prompt 可换行或截断。
8. 下载一致：后续应统一使用 Blob 下载，且 loading 只出现在当前点击按钮上。

## 3. 视觉方向

整体视觉采用“安静的生成工具”方向：低噪声背景、清晰边界、少量高亮色、充足留白、紧凑但不拥挤的表单密度。

- 色彩：沿用现有主题系统，主按钮使用单一强调色；警告和错误只在需要处理时出现。
- 字体：系统字体优先，标题克制，正文和控件保持高可读性。
- 卡片：减少大面积装饰卡片，生成页只保留输入面板和历史面板两类容器。
- 图标：可用在任务、主题、语言、历史布局、刷新、删除、下载等高频短按钮中；图标按钮必须有可访问名称。
- 按钮文字：所有按钮文字视觉居中，图标和 loading 不得造成文字偏移。

## 4. 信息架构

现有路由保持不变：

- `/`：首页，提供产品定位和进入图片/视频生成的主入口。
- `/image`：图片生成，支持文生图、图生图、多图合成。
- `/video`：视频生成，支持文生视频、图生视频、多图视频、关键帧动画。
- `/privacy`：隐私政策。
- `/terms`：服务条款。
- `/contact`：联系页。
- `*`：404。

全局外壳：

- 顶部导航：品牌、首页、图片生成、视频生成、任务按钮、设置入口。
- 设置面板：主题、语言、健康状态灯。健康状态保留 `title` 和 `aria-label`，页面上不展示系统连接说明。
- 任务抽屉：显示当前标签页任务队列、进度、错误、结果、移除和重试。
- 页脚：保留合规页入口，生成页可使用宽版布局。

## 5. 页面规划

### 5.1 首页

首页目标是快速分流，而不是承载完整说明。

- Hero 保留一句定位、两个主 CTA：生成图片、创建视频。
- 功能区只展示图片生成、视频生成、本地历史/预览下载三类能力。
- 使用流程压缩为输入、上传、参数、下载四步。
- 透明披露文案不在首页堆叠展示，可改为页脚合规入口或一条极短提示。
- 移动端 CTA 纵向排列，按钮宽度一致，高度不少于 44px。

### 5.2 图片生成页

布局保持“左输入、右历史”。桌面端 1280px 以上建议 6:4 配比，输入区优先。

- 模式：文生图、图生图、多图合成使用单个选择器，不再增加模式说明卡。
- 上传：仅在图生图和多图合成显示；图生图最多 1 张，多图最多 8 张。
- Prompt：主输入区固定可见，Prompt 优化按钮使用 `✦` 或短按钮贴近输入框。
- 图片理解：仅有参考图时出现；多图时只用简短方式标记“分析首张”。
- 参数：尺寸、输出格式、重置默认值保持在 Prompt 下方。
- 提交按钮：全宽或显著宽度，显示生成、合成、还需图片数、队列满、提交中、已接受等短状态。
- 提交成功：清空 prompt、参考图和草稿，保留尺寸偏好，并刷新历史。

### 5.3 视频生成页

视频页与图片页保持同构，减少用户学习成本。

- 模式：文生视频、图生视频、多图视频、关键帧动画使用选择器。
- 上传：非文生视频显示；图生视频最多 1 张，多图和关键帧最多 8 张。
- Prompt 优化与图片理解复用图片页交互。
- 参数：负向 Prompt、时长帧数、帧率、分辨率、seed 按频率排序；预计时长作为轻量 hint。
- 视频耗时提示应短，不保留冗长免责声明。
- 提交成功后清空 prompt 和参考图，保留视频模式、负向 Prompt、帧数、帧率、尺寸和 seed 偏好。

### 5.4 任务队列

任务队列是跨页面能力，由全局任务按钮和右侧抽屉承载。

- 顶部任务按钮显示 `active/max`，窄屏优先图标化。
- 抽屉宽度在桌面不遮挡全部内容，移动端可全宽滑出。
- 任务卡展示类型、prompt 摘要、尺寸、进度、队列位置、结果和错误。
- 最大进行中任务数为 3，提交节流为 1200ms；队列满时生成按钮禁用。
- 查询失败且已有服务端任务 ID 时提供重试查询；完成或失败后写入本地历史。

### 5.5 本地历史

本地历史是当前产品的主要恢复能力，不扩展为云相册。

- 图片和视频分别显示最近本地记录，分类读取最多 100 条，总量最多 200 条。
- 历史工具栏包含数量、搜索、状态筛选、排序、列表/瀑布流。
- 窄面板下工具栏必须 `flex-wrap`，可将标签图标化，搜索独占一行，避免横向滚动。
- 点击历史记录可回填模式、prompt、参数和历史保存的 URL 参考图；本地文件不回填。
- 输入区已有内容时，回填前保持确认，避免覆盖用户草稿。
- 移动端瀑布流强制单列。

### 5.6 上传

- 上传入口包含本地图片选择和公网 URL 添加。
- 本地文件按钮、URL 输入和添加按钮在移动端可换行，控件高度不少于 44px。
- 缩略图展示来源标签、删除按钮和预览入口。
- 超过最大数量时不新增；超过单文件大小上限时显示短错误。
- URL 交互提示使用 `https://` 公网 URL，但不在前端增加与当前实现不一致的强校验承诺。

### 5.7 Prompt 优化

- 空 Prompt 时给出“先输入提示词”一类短提示。
- 优化中按钮显示局部 loading，禁止重复点击。
- 成功后直接回填输入框，并短暂标记完成。
- 失败时显示靠近按钮或输入框的错误，不阻断用户继续编辑。

### 5.8 图片理解

- 仅在有参考图时显示。
- 图片来源优先第一张本地文件，其次第一条 URL。
- 结果区提供“填入 Prompt”和“复制”两个动作。
- 复制失败静默处理，用户仍可手动选择文本。
- 多图场景用短标签说明只分析第一张，避免重复解释。

### 5.9 预览

- 图片和视频共用预览弹窗。
- 支持遮罩关闭、关闭按钮和 `Escape`。
- 弹窗内容区域最大化展示媒体，底部只保留下载和关闭。
- 视频使用 `<video controls playsInline preload="metadata">`。
- 预览弹窗下载后续应替换为 `BlobDownloadButton`，保持当前按钮独立 loading。

### 5.10 下载

- 目标交互统一为 `fetch -> Blob -> objectURL -> a.click()`。
- loading 只出现在当前点击的下载按钮，不影响同一结果的其他按钮。
- 下载失败使用短错误展示 HTTP 状态或错误消息。
- 文件名沿用当前规则：图片 `agnes-image-{taskId}-{index}.png`，视频 `agnes-video-{taskId}.mp4`。
- 任务抽屉图片结果和预览弹窗下载是后续统一 Blob 下载的重点。

### 5.11 设置

- 主题：`auto`、`default`、`light`、`dark`，使用图标和选择器。
- 语言：中文/英文直接切换，按钮可显示 `中` 或 `EN`。
- 健康状态：只显示状态灯；详细状态放在 `title`、`aria-label`。
- 移动端设置面板折叠在菜单内，打开后控件纵向排列。

### 5.12 通知

- 通知默认关闭，不在页面加载时主动打扰用户。
- 若后续增加可见开关，应由用户显式开启后再申请浏览器权限。
- 关闭通知时需要立即关闭当前网页已弹出的通知实例。
- 通知内容只用于任务完成或失败，不扩展为营销提醒。

### 5.13 合规页

隐私政策、服务条款和联系页保持独立路由，通过页脚进入。

- 合规页正文使用窄版阅读宽度，信息清楚但不过度强调。
- 联系页上线前必须替换占位邮箱。
- 首页和生成页不重复堆叠免责声明，只保留必要链接或短提示。
- 隐私页继续说明 `localStorage`、Agnes API、ImgBB、本地历史和无统计追踪。

### 5.14 404

- 保留未找到提示。
- 提供返回首页、图片生成、视频生成三个恢复入口。
- 移动端按钮纵向排列，视觉优先级为首页和两个生成入口。

## 6. 核心组件规范

- 顶部导航：桌面横向，移动端品牌与任务按钮优先，导航和设置可折叠。
- 设置面板：控件高度不少于 44px，select 图标与右侧边框留足间距，垂直居中。
- Prompt 输入：至少 4 行，移动端宽度 100%，优化按钮不遮挡输入文本。
- 上传区：按钮、输入框、添加按钮可换行；缩略图删除按钮命中区不少于 44px。
- 参数控件：相关参数两列排列，窄屏单列；重置按钮文字居中。
- 生成按钮：主按钮居中显示文字和 spinner，禁用态仍可读。
- 历史工具栏：数量、搜索、筛选、排序、布局都可换行；窄面板允许图标化标签。
- 结果卡：优先展示结果缩略图/视频，其次是状态和操作；长 prompt 使用截断并提供 `title`。
- 预览弹窗：焦点应进入弹窗，关闭后返回触发元素；遮罩不造成页面横向滚动。
- 下载按钮：统一使用 Blob loading 状态，loading 文案和 spinner 不造成布局跳动。
- 状态提示：错误靠近发生位置；全局错误只用于跨组件问题。

## 7. 状态反馈

- 提交前：按钮根据 prompt、图片数量、队列容量和节流状态给出短标签。
- 提交中：任务进入抽屉，显示提交中或进度条。
- 排队中：有队列位置时显示短文本。
- 运行中：进度条为主，减少解释文字。
- 完成：展示结果、预览和下载。
- 失败：展示可理解错误；可重试查询时给出重试按钮。
- 健康异常：状态灯警告，不用大段系统连接文案占据页面。
- 限流：显示重试等待时间和限制数量，避免技术堆栈信息。

## 8. 空态、错误态、加载态

- 历史加载中：使用轻量 skeleton 或短文本。
- 无历史：提示“暂无本地历史”，并给出生成入口语义，不增加教程。
- 筛选无结果：显示无匹配记录，并保留清空搜索/切换筛选的路径。
- 上传为空：非必须模式不显示上传区；必须模式通过生成按钮提示还需图片数。
- 图片理解无图：组件不渲染。
- 结果缺失：完成但无图片或视频 URL 时显示短错误。
- Blob 下载失败：按钮恢复可点击，并显示短错误或 alert 作为当前兼容方案。

## 9. 响应式规则

- 1280px 以上：生成页采用 6:4 输入/历史比例。
- 768px-1279px：可继续双列，但历史面板缩窄时工具栏必须换行。
- 320px-767px：页面单列，顶部导航折叠，历史在输入之后。
- 320px-480px：禁止任何横向溢出；URL、prompt、文件名、错误消息必须换行或截断。
- 所有触控控件高度不少于 44px，图标按钮命中区也按 44px 设计。
- 抽屉在移动端全宽或接近全宽，关闭按钮固定可见。

## 10. 可访问性

- 所有图标按钮必须有 `aria-label` 或可见文本。
- 任务抽屉和预览弹窗使用语义化 `dialog`/`aside`，关闭按钮可键盘访问。
- 颜色状态必须配合文本、`title` 或 `aria-label`，不能只靠颜色表达。
- 表单控件使用明确 label；错误信息靠近控件。
- `Escape` 可关闭预览弹窗；后续抽屉也应考虑键盘关闭。
- loading 按钮设置 `disabled`，避免重复提交。

## 11. 微交互

- 提交成功反馈持续约 1.4 秒，之后恢复常态。
- Prompt 优化、图片理解、下载使用局部 spinner。
- 历史筛选输入使用延迟过滤，减少输入卡顿。
- 卡片删除、刷新、回填使用短反馈，避免整页刷新。
- 主题和语言切换即时生效，并同步文档属性。

## 12. 本地持久化体验

继续使用 `agnes:` 前缀：

- 草稿：图片和视频 prompt、模式、参数、URL、本地文件 Data URI。
- 偏好：主题、语言、图片尺寸、视频模式、视频尺寸、帧数、帧率、负向 Prompt、seed。
- 历史：图片/视频历史记录、搜索词、状态筛选、排序和布局。
- 通知：通知启用状态和权限状态。

所有本地存储读写必须捕获异常。隐私模式、配额不足或存储不可用时，不影响生成、上传和下载主流程。

## 13. 分阶段实施优先级

### P0：核心生成与移动端稳定

- 清理首页和生成页冗余引导/免责声明文案。
- 确认按钮文字居中，所有触控控件高度不少于 44px。
- 修复 320px-480px 横向溢出风险。
- 生成页桌面端按 6:4 强化主功能区。
- 历史工具栏在窄面板下换行和图标化。

### P1：状态与组件一致性

- 统一图片/视频页的 Prompt、上传、图片理解、提交反馈布局。
- 任务抽屉、历史卡、结果卡的状态语言统一为短标签。
- 健康状态只保留状态灯，详细原因走辅助属性。
- 合规页保留准确内容，但从首页和生成页移除重复披露。

### P2：下载与预览统一

- 任务抽屉图片下载、预览弹窗下载统一迁移到 Blob 下载。
- 下载按钮只显示当前按钮 loading。
- 预览弹窗补齐焦点管理和下载错误反馈。

### P3：可访问性与持久化打磨

- 补齐图标按钮可访问名称和键盘路径。
- 抽屉支持 `Escape` 关闭和焦点返回。
- 验证所有 `localStorage` 异常不影响主流程。
- 通知改为显式开关后再申请权限。

## 14. 验收标准

- `UI_DESIGN_PLAN.md` 覆盖首页、图片生成、视频生成、任务队列、本地历史、上传、Prompt 优化、图片理解、预览、下载、设置、通知、合规页和 404。
- 规划未引入账号、支付、分享、云相册、团队协作、统计追踪或权限管理。
- 320px、375px、414px、480px 宽度下无横向滚动。
- 所有触控控件高度不少于 44px。
- 历史工具栏在桌面窄面板和移动端均可换行，不挤压出容器。
- 主生成按钮、下载按钮、重置按钮和图标按钮文字/图标视觉居中。
- 下载交互的后续目标明确为 Blob loading，且 loading 只作用于当前按钮。
- 首页和生成页不重复展示冗余引导、状态解释或免责声明。
- 任务队列上限、1200ms 节流、本地历史限制、上传数量限制和本地持久化键名与现有实现一致。
